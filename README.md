# L2 Geodata Toolkit

Инструмент для работы с геодатой Lineage 2: конвертация между форматами
PTS (`XX_YY_conv.dat`) и L2J (`XX_YY.l2j`), браузерный просмотрщик,
рендер в картинку, валидация и сверка наборов. Все длительные
операции показывают прогресс-бар с оценкой оставшегося времени.

Требования: **Python 3.8+**, только стандартная библиотека (никаких зависимостей).

## Запуск

```bash
python3 geotool.py              # интерактивное меню
```

Или подкомандами:

```bash
python3 geotool.py convert <папка_pts> -o <папка_l2j> [-y]   # PTS → L2J
python3 geotool.py l2j2pts <папка_l2j> -o <папка_pts>        # L2J → PTS
python3 geotool.py view <папка> [--port N]                   # просмотрщик
python3 geotool.py diff <папка_A> <папка_B> [--region XX_YY] # сравнение наборов
python3 geotool.py generate <клиент> -o <папка> [--region ...] [-j N] [--terrain-only]
python3 geotool.py check <папка|файлы...>
python3 geotool.py verify <папка1> <папка2>   # сверка PTS ↔ L2J (порядок любой)
```

## Просмотрщик (`view`)

Поднимает локальный веб-сервер (127.0.0.1) и открывает браузер:

- слева список регионов (жёлтый бейдж — плоская заглушка);
- карта региона 256×256 блоков, палитра нормализована на диапазон высот;
- **колесо мыши** — зум ×1…×32 к точке под курсором, кнопки **+/−** — к центру;
- **перетаскивание ЛКМ** — панорамирование;
- **клик по блоку** → инспектор: ячейки 8×8, выбор слоя;
- **клик по ячейке** → слои с высотами и NSWE-проходимостью;
- селектор **«Слой»** в верхней панели — 1 (поверхность), 2, 3… на весь регион
  (мосты, этажи, подземелья); блоки без выбранного слоя гаснут;
- с зума **×8** закрытые NSWE-направления рисуются красными гранями (стены/обрывы);
- кнопка **⬇** на карте — скачать PNG региона 2048×2048 (пиксель = ячейка,
  с учётом выбранного слоя).

Просмотрщик держит в памяти до двух распарсенных регионов (~0,5 ГБ RAM
на тяжёлых регионах) — на машинах с 8 ГБ лучше не открывать параллельно
несколько экземпляров.

## Сравнение наборов (`diff`)

Консольное сравнение двух папок геодаты — например, нового скачанного пака
с текущим, прежде чем ставить:

- **сводный отчёт** — все общие регионы, отсортированные по расхождению
  поверхностей (быстрая детерминированная выборка), процент совпадения,
  максимум |Δh|, кто отсутствует в какой папке;
- **`--region XX_YY`** — полный поячеечный разбор региона: точная статистика
  высот **и проходимости NSWE** (совпадение, расхождения по направлениям),
  очаги расхождений с мировыми координатами и ASCII-карта 32×32.

## Генерация из клиента (`generate`)

Строит геодату напрямую из файлов клиента L2 (нужны папки `Maps`,
`Textures`, `StaticMeshes`). Без UnrealEd и HEX-редакторов.
Генерирует **оба варианта карт** по папкам: `out/main/` (из `XX_YY.unr`,
все регионы) и `out/classic/` (из `XX_YY_Classic.unr`, только
переработанные зоны). Классик-сервер собирается как main + classic
поверх.

Ядро — **heightfield в стиле Recast** (тот же принцип, что у рейкаста
сверху вниз, но эффективно через растеризацию в спаны):

- дешифровка Lineage2Ver111/121, парсинг UE2-пакетов (чистый Python);
- **рельеф**: G16-heightmap из `T_XX_YY.utx` + TerrainInfo из `XX_YY.unr`
  (позиция/масштаб), высоты приведены к принятой в L2J конвенции; рельеф
  проходим всегда (в т.ч. под «дырами» QuadVisibilityBitmap — в застройке
  это зоны замены террейна мостовой, а не провалы);
- **статик-меши + BSP-модель уровня** (здания, мосты, соборы, подземелья):
  по-материальная коллизия (`Materials[i].EnableCollision`, как в клиенте —
  кроны деревьев не участвуют), UDK-правило флагов актора;
- **классификация граней по нормали** (пол ≤ 40°):
  грань вверх → пол-спан, грань вниз → потолок/дно (полом не становится),
  что конструктивно убирает паразитные нижние слои от объёмных мешей;
- **слои с клиренсом персонажа**: пол проходим, если над ним до ближайшей
  поверхности есть зазор в рост (иначе под нависшей геометрией не встать);
  близкие по высоте полы сливаются (порог округления);
- **фильтр зонных/водных плоскостей**: горизонталь размером с регион — не
  пол, а плоскость зоны/воды, в геодату не идёт;
- **NSWE по перепаду высот соседей** (как в реальной геодате): направление
  закрыто, если у соседа нет слоя в пределах шага (подъём > 16 закрыт,
  спуск свободен до 96). Явных стен-блокировок нет — заборы/перила/двери
  в L2J закрывает сервер (FenceData/DoorData), геодате они не нужны;
- отсев недостижимых с рельефа островков (кроны, верхушки валунов);
- параллельная генерация: по умолчанию ≈ половина ресурсов (половина ядер, но не больше, чем влезает в половину ОЗУ при ~2.5 ГБ на воркер — комп остаётся рабочим и не уходит в своп); явное число — `-j N`.

Каждый результат валидируется парсером, повторяющим загрузчик движка.
Проверить качество на своих данных: сравни с имеющейся геодатой через
`diff` (высоты и NSWE), а спорные и ключевые зоны обязательно пройди
в игре — финальный судья всегда клиент, а не число.

Известные ограничения: двери и заборы в геодату не пишутся сознательно —
их проходимость закрывает сервер (DoorData/FenceData); вода — записывается
дно (как у всех генераторов); сырые CSG-браши не растеризуются (пост-CSG
геометрия берётся из BSP-модели).

## Форматы и гарантии

Спецификации обоих форматов и расшифровка 18-байтового заголовка PTS —
в докстринге `geolib/formats.py`. Каждый результат конвертации валидируется
парсером, повторяющим логику загрузчика движка.

Гарантии точности:

- **L2J → PTS → L2J: байт-в-байт** — типы блоков сохраняются как в исходнике.
  Единственное исключение — вырожденный multilayer-блок, где все 64 ячейки
  одно-слойные: по спецификации PTS он неотличим от complex и кодируется
  как complex (семантика для движка идентична; в реальной геодате таких
  блоков не встречается).
- **PTS → L2J → PTS: байт-в-байт** — маркер multilayer-блока восстанавливается
  как сумма слоёв, заголовок — по счётчикам блоков.
  Проверено на реальных паках. Теоретическое исключение: PTS с flat-блоком,
  у которого min ≠ max — L2J хранит одну высоту, min был бы потерян;
  на практике в реальных паках всегда min = max (проверено на 13,5 млн блоков).

Имена файлов — строго канон: `XX_YY_conv.dat` и `XX_YY.l2j`. Файлы с иными
именами, но подходящие под маску (`*_conv.dat`, `*.l2j`), пропускаются
с предупреждением и кодом выхода 3; файлы вне масок (например `XX_YY.dat`)
не рассматриваются вовсе.

## Структура

```
geotool.py          точка входа (меню + CLI)
geolib/
  formats.py        парсеры/кодировка L2J и PTS
  convert.py        конвертеры в обе стороны
  checks.py         валидация и сверка наборов
  render.py         палитра высот + PNG-рендер (для просмотрщика)
  unreal.py         дешифровка и парсер UE2-пакетов клиента
  meshes.py         парсер UStaticMesh + трансформации акторов
  generate.py       генерация геодаты из клиента
  diff.py           сравнение двух наборов
  viewer.py         HTTP-сервер просмотрщика
  page.py           HTML/JS страница
  ui.py             оформление CLI
```
